# 📈 OGDC Analytics Dashboard


Interactive frontend for the full OGDC stock price analysis project covering EDA, ANOVA, Distribution Fitting, Multiple Regression, ARIMA, GARCH, ML Models, Backtesting, Sentiment Analysis, and Enhanced ML.

## Setup — Step by Step

### Step 1 — Prerequisites
Make sure you have Node.js installed (version 18 or higher):
```bash
node --version   # should print v18.x.x or higher
npm --version    # should print 9.x.x or higher
```
Download Node from https://nodejs.org if needed.

### Step 2 — Install dependencies
```bash
# Navigate into the project folder
cd ogdc-analytics

# Install everything listed in package.json
# This creates the node_modules/ folder (takes ~30 seconds)
npm install
```

### Step 3 — Add your data files to /public
Copy these 3 JSON files (generated by your Python scripts) into the `public/` folder:
```
public/price_data.json
public/kpis.json
public/annual_stats.json
```

### Step 4 — Add your chart PNGs to /public
Copy all 54 PNG chart files into the `public/` folder.
They should be named exactly as your Python scripts saved them:
```
public/stat_01_eda_panel.png
public/stat_02_scatter_matrix.png
public/stat_03_anova.png
public/stat_04_distribution_fitting.png
public/stat_05_regression.png
public/stat_06_arima.png
public/stat_07_nonparametric.png
public/trend_01_moving_averages.png
public/trend_02_bollinger_bands.png
... (all 54 PNGs)
```

### Step 5 — Run locally
```bash
npm run dev
```
Opens at http://localhost:5173 — hot-reloads on every save.

### Step 6 — Build for deployment
```bash
npm run build
```
Creates a `dist/` folder of optimised static files. This is what you deploy.

## Deployment Options

### Option A — Vercel (Recommended, easiest)

1. Push this project to GitHub
2. Go to https://vercel.com → "New Project"
3. Import your GitHub repo
4. Vercel auto-detects Vite — click Deploy
5. Done. You get a URL like `ogdc-analytics.vercel.app`

Everything is configured in `vercel.json` already.

### Option B — Netlify

1. Push to GitHub
2. Go to https://netlify.com → "Add new site" → "Import from Git"
3. Connect repo — Netlify reads `netlify.toml` automatically
4. Click Deploy

Or drag-and-drop the `dist/` folder onto https://app.netlify.com/drop

### Option C — GitHub Pages

Add this to `vite.config.js`:
```js
base: "/your-repo-name/",
```
Then:
```bash
npm run build
# Push the dist/ folder to the gh-pages branch
npx gh-pages -d dist
```

## Data Files Explained

| File | Generated by | Contents |
|---|---|---|
| `price_data.json` | `ogdc_loader_cleaner.py` | Full OHLCV + SMA20/50/200 + BB + HV20 |
| `kpis.json` | Python one-liner | Latest price, total return, Sharpe, max DD, HV |
| `annual_stats.json` | Python groupby | Mean, std, min, max return per year |

All JSON files are loaded by the app via `fetch()` in a `useEffect` hook.
They live in `/public` so Vite serves them as static assets.

## Common Issues

**Charts not showing?**
Make sure your PNG files are in the `public/` folder, not `src/`.
File names must match exactly (case-sensitive on Linux/Mac).

**Blank page after `npm run build`?**
Check the browser console for errors.
Most common cause: a missing JSON file in `public/`.

**Fonts not loading?**
The app loads Space Mono from Google Fonts.
Requires internet connection. Works fine in deployment.
